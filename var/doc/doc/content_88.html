Syntax
<float> ::= <int> '.' {<int>}? {'e' | 'E' {'+' | '-'}? <int>}?

Description

This form represents an floating point literal. There is no negative floating point literal.

Semantics

The result of evaluating a float literal will have type float and the specified value. A negative floating point value can be produced by applying the - prefix operator to a floating point literal.

Parsing

Parsing an float literal results in a xdom:float tree node.

Examples


   2.3 * 5.0E20
