Since @x:Act scripts can easily generate HTML, it is natural to run the @x:Act interpreter from a web server. The simplest way to do this is to run as a CGI. The Windows file xact/dev/win/runc.bat (or the Macintosh file xact/dev/mac/runc)is a sample command for running @x:Act as a CGI. Copy this file to a CGI directory and edit the path to definitions.

Due to a bug in Apache 2 on Windows (loss of STDOUT from child programs), we know no way to run @x:Act under Apache on Windows. We recommend instead using the servlet version of @x:Act on Windows. This problem does not occur with Unix versions of Apache 2 or with other Windows web servers.
