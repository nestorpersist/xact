Description

The stream type is is used for high-level multithread synchronization. One thread sends strings to the stream, while another thread receives the concatenated results.

Type stream

@x:self @x:func send(val~string)~void

Self Function send

@x:func send(val~string)~void
val A string value to be sent.

Semantics

Sends a value to the queue. Waits if queue is full.

Self Function close

@x:func close()~void

Semantics

This function should be called after all calls to send to close the stream. The receive call will not complete until close is called.

Self Function receive

@x:func receive()~string

Semantics

Receives and returns a value from the queue. Waits if the queue is empty.

Type Function sys:Eq

@x:func sys:Eq(a~stream,b~stream)~boolean
a first parameter
b second parameter

Semantics

If a and b are the same stream, then the result is true, else the result is false.

Type Function sys:Ne

@x:func sys:Ne(a~stream,b~stream)~boolean
a first parameter
b second parameter

Semantics

If a and b are the same stream, then the result is false, else the result is true.

Self Function sys:Init

@x:func sys:Init(%L)~void
%L This can be either empty or a single int size.

Semantics

Initializes a new stream object of size specified by the count (or size 1 if the count is not specified). This count is the number of sent strings that can be held in the stream.

Self Function toString

@x:func toString()~string

Semantics

The result is the string "stream".
