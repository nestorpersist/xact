@x:Act scripts can be parsed, manipulated, and then unparsed. This makes it easy to write tools that work on @x:Act scripts. Since XML is a subset of the @x:Act syntax, it is also easy to write tools for XML.

The result of parsing an @x:Act script is a xdom tree.

The following example counts the number of commands in itself. It reads in a copy of itself, parses that copy to produce a xdom tree, then walks that tree using a recursive iterator.


@x:use library:unparser;
@x:use library:iterator;
@x:use library:path;

@x:func elements[x~xdom](act) {
   @x:if x?xdom:element {
      act(x);
      @x:for x1 := x.all#body {
         elements[x1](act)
      }
   }
}
@x:var cnts := rec();
@x:const fname := pathToFile("example:cntTag.xact");
@x:const txt := fname.read#file();
@x:const root := Parse(txt);
@x:for e := elements[root] {
   @x:if e != root {
      @x:const tag := e.tag;
      @x:var cnt := Dot(cnts,tag,rec);
      @x:if cnt == null { cnt := 0 }
      Dot(cnts,tag,rec) := cnt + 1
   }
}
Unparse(cnts); eol

The xdom:element type is used to represent @x:Act commands and XML elements. It also has views that enable it to be used as a record (rec) or array (array).

The record dot qualification operation r.x#rec has a generalized form Dot(r,"x",rec) that allows the component name to be a computed value.
