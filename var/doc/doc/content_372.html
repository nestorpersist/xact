Description

This function will unparse @x:Act scripts and the subset of XML supported by @x:Act syntax. It also handles conversions to Pure XML.

Function Unparse

@x:func Unparse(tree~xdom,indent=indent1~int := 0,incr=incr1~int := 3,sugar=sugar1~boolean := true,graph=graph1~boolean := false,html=html1~boolean := false,partial=partial1~enum["","start","body","end"] := "",callBack=callBack1 := null)~string
tree parse tree to be unparsed.
indent=indent1 (optional) number of character positions to skip before starting each line of output.
incr=incr1 (optional) number of additional character positions to indent for each nested level.
sugar=sugar1 (optional) whether to show sugared forms in the output.
graph=graph1 (optional) whether the input should be treated as a graph with sharing and cycles (rather than a tree). If cycles are present and graph=false, then Unparse will fail (with a stack overflow). For more information see Graphs and the graph example.
html=html1 (optional) output should be XHTML (rather than plain text).
partial=partial1 (optional) if tree has type xdom:element, then this parameter controls which parts of it should be output: only the start tag ("start"), only the body ("body"), only the end tag ("end"), or all parts (""). each nested level.
callBack=callBack1 (optional) see below. This parameter is used to generate @x:Act documentation. If allows XHTML documentation links to be added for tags, names, infix ops, prefix ops.

Semantics

Returns the resulting unparsed text.

If the callBack parameter is present, it should have the following form.

Function callBack

@x:func callBack(kind~enum["preop","inop","name","tag"],name~string)~string
kind kind of text given by name.
name text of name

Semantics

Returns the marked-up (for @x:Act documentation an html a tag is added) form of name. If just name is returned, the call-back has no net effect.
