Description

The queue type is is used for high-level multithread synchronization. One thread sends values to the queue, while another thread receives those values.

The queue examples shows how a type with the same semantics as queue can be implemented using locks.

Type queue

@x:type queue

@x:self @x:func send(val)~void
@x:self @x:func receive()
@x:type @x:func sys:Eq(a~queue,b~queue)~boolean
@x:type @x:func sys:Ne(a~queue,b~queue)~boolean
@x:self @x:func sys:Init(count~int)~void
@x:self @x:func toString()~string

Self Function send

@x:func send(val)~void
val A value to be sent.

Semantics

Sends a value to the queue. Waits if queue is full.

Self Function receive

@x:func receive()

Semantics

Receives and returns a value from the queue. Waits if the queue is empty.

Type Function sys:Eq

@x:func sys:Eq(a~queue,b~queue)~boolean
a first parameter
b second parameter

Semantics

If a and b are the same queue, then the result is true, else the result is false.

Type Function sys:Ne

@x:func sys:Ne(a~queue,b~queue)~boolean
a first parameter
b second parameter

Semantics

If a and b are the same queue, then the result is false, else the result is true.

Self Function sys:Init

@x:func sys:Init(count~int)~void
count A queue size.

Semantics

Initializes a new queue object of size specified by size.

Self Function toString

@x:func toString()~string

Semantics

The result is the string "queue".
