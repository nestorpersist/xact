Description

This command encloses a set of @x:thread commands to provide multi-thread fork/join semantics.

Command Syntax

'@x:fork'

No attributes, options or body are permitted.

Execution Semantics

The result of executing a @x:thread command is the result of executing its body.

After the body execution completes, the current thread waits until all threads created by immediately enclosed @x:thread commands finish executing.

Examples


   @x:fork {
      @x:thread t1 { subtask1() }
      @x:thread t2 { subtask2() }
      maintask()
   }

See IncrParse example and queue example.
