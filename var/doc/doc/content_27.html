By default, commands whose tags have no namespace or a namespace other than x: have render semantics by default.

The result of executing a command with render semantics is a string whose value is the string representation of the command.

For example, consider a <script> containing only XML (with no tags having a x: namespace). The result of executing that script is similar to the effect of calling Unparse on the xdom of the script. The main difference is that the pretty printing of spaces will be different. While Unparse indents nested elements, render semantics does no indentation for nested elements.

Render semantics, however, is more than just an unparser. No only can @x:Act (x:) commands contain rendered commands, but rendered commands can contain name uses and @x:Act commands that parameterize the rendered result.

The is flexibility in the generation of space characters in the result of rendered commands. If precise control of spacing is needed, an alternative is the use @x:Act @x:tag command that renders an specified xdom tree.
