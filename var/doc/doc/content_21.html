A <name> can be either an <id> or can be an <id> followed by a list of expressions enclosed in [] brackets. For name binding purposes a name without brackets is considered to be a different name than a name with brackets. Two names with the same id and bracketed expression lists are considered to be the same name for name binding purposes even if the number of expressions and their values are different.

The are two forms of <id>. A <basic id> is a Java-like identifier. An <extended id> is an XML-like identifier and permits more characters than the <basic id>

@x:Act has no reserved words, so all <id>s are permitted. There are however many predefined names. Predefined names can be redefined.

For the x: namespace, names are defined in several places.

  • The @x:var command define a variable name in the enclosing scope.
  • The @x:const command defines a constant name in the enclosing scope.
  • The @x:func command defines a function name in the enclosing scope and name and regular parameter names within the function scope.
  • The @x:type command defines a type name in the enclosing scope and name parameter names within the type scope.
  • The @x:view command defines a view name in the enclosing scope and name parameter names within the view scope.
  • The @x:for command defines iteration names within the for scope.
  • The @x:thread command define a thread name in the enclosing scope.
  • The name self is automatically defined within the @x:self command.
  • The @x:module command defines exported names in the enclosing scope.
  • The @x:use command defines imported names in the enclosing scope.

A <name> within an <exp> that is not a definition of that name is a use of that name. Each extended id form of <escape> is also a use of that extended id name.
