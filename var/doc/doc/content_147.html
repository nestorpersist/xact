Description

The xdom:element type is used to represent @x:Act commands and XML element nodes within an xdom tree.

Subscripting of an xdom:element provides array access. The ith element will be the value of the ith unnamed attribute.

Views of xdom:element include rec, attr, body, bodyrec, @icode{array}/>, and list. These views provide subscripting based on other views of the list of attributes and body subtrees.

Self Var tag

@x:var tag~string

Semantics

The command name / XML element tag.

Self Var space

@x:var space~string

Semantics

The part of tag before the first : or "" if there is no :.

Self Var base

@x:var base~string

Semantics

The part of tag after the first : or the entire tag if there is no :.

Self Const size

@x:const size~int

Semantics

The number of array elements (unnamed attributes).

LHS/RHS Self Function sys:Subscript

@x:func sys:Subscript(index~int)
index The index of the array element (unnamed attribute) to be selected. For element e this must be in the range 1..e.size.

Semantics

The indexth array element (unnamed attribute) value.

Self Function insert

@x:func insert(val,pos~int := self.size)~void
val a value to be inserted
pos (optional) the value must be in the range 0..self.size.

Semantics

Inserts the new value at the specified position. If the position is 0, the value is inserted at the beginning of the array. Otherwise, the value is inserted after the pos element.

Self Function delete

@x:func delete(pos~int)~void
pos (optional) for element e, the value must be in the range 1..e.size.

Semantics

The element at the specified position is deleted from the array.

Self Const all

@x:const all~func

Semantics

An iterator function that iterates over all the array elements.
