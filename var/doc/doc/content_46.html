@x:Act provides a number of incremental execution features that allow processing of data that is too large to all fit in main memory at the same time. Incremental parsing in one of these features.

There are two major kinds of XML parsers, those that build an entire DOM tree into main memory and those that don't build a tree but instead generate events (SAX). The @x:Act Parse function not only provides both these modes but also provides a way to mix the two techniques.

In the simple case the Parse function just builds an xdom tree.

For incremental processing, the Parse is passed an instance of a user-defined type that contains @x:self functions that serve as event handlers. The endAttr function is called after each attribute tree is constructed and is passed that tree. The endChild function is called after each child of a command is constructed and is passed that tree. In each case the function can then decide whether to keep, discard, or replace that tree. By discarding subtrees as they are processed, parsing becomes incremental.

See the IncrParse example.
