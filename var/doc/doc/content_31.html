There are a number of simple built-in types.
  • The type of null is void. The null value is used for expressions with "no" result.
  • @x:Act strings are based on Java strings and use the same Unicode 2-byte character set. String values have type string. Views of string include fold, boolean, date, file, and enum. There are <string> literals. Literal strings also appear as <xml> text. Tree nodes with type xdom:string hold string values.
  • @x:Act integers are based on Java longs and have a 64 bit signed representation. Integer values have type int. There are <int> literals. Tree nodes with type xdom:int hold integer values.
  • @x:Act floats are based on Java doubles and have a 64 bit representation. Float values have type float. There are <float> literals. Tree nodes with type xdom:float hold float values.
