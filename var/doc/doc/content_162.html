Description

View of xdom:element that allows named attributes to be treated as a record. Record values are stored as follows.

string value stored in a xdom:string node
int value stored in a xdom:int node
float value stored in a xdom:float node
any other value stored in a xdom:value node

View rec

@x:view rec

@x:view @x:func sys:Is(v~boolean)~boolean
@x:view @x:func sys:Make(%L)~rec
@x:self @x:func sys:Dot[name~string]

View Function sys:Is

@x:func sys:Is(v~boolean)~boolean
v value to test

Semantics

The result is v?xdom:element.

View Function sys:Make

@x:func sys:Make(%L)~rec
%L a list of named values

Semantics

The result is a new record initialized to the specified named values. The tag of the resulting xdom:element is "rec".

LHS/RHS Self Function sys:Dot[ ]

@x:func sys:Dot[name~string]
name the name of the record value to be selected

Semantics

The result is the record component with the specified name. On the rhs, if the named component does not exist the result is null. On the lhs, if value being assigned is null, any component with name name is removed from the record. On the lhs, if the named component does not exist, it is added.
