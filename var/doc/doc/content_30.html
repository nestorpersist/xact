@x:Act is dynamically typed. Every value has a type, but multiple executions of a single expression may yield values with different types.

The Type function returns the type of a value. The ? infix operator tests if a value has a specified type. The ~ infix operator asserts that a value has a specified type. The ~ infix operator can also be used on @x:var declarations, @x:const declarations, and @x:func declarations formal parameter and result specifications.

A prefix form of the ~ operator is also available. It asserts that the following expression has value true.

@x:Act has a number of predefined types. The @x:type command is used to define new types.

A type controls the set of values of an object of the type. It also controls the set of operations that can be applied to the type and to objects of the type.

A view is similar to a type, except it is specified as part of operations on values. For example, the string == infix operator compares two strings for equal characters. The view operation ==#fold infix operator compares two strings after case-folding.

@x:Act has a number of predefined views. The @x:view command is used to define new views.

Views can be used for subsetting. For example, the boolean view is a view of strings that allows only two values "true" and "false".

Views can also be used for supersetting. For example, the view TypeOr[int,string] contains both type int and type string values.

@x:Act does not support type inheritance. Views can be used in many places where inheritance is used in other languages.
