When building a web site, a way is needed to provide links from a page generated by one script to another script. @x:Act provides a rich set of facilities for this.

@x:Act scripts are stored in a scripts directory tree. The scriptDir option gives the path to this directory. @x:Act scripts can be placed in this directory or in subdirectories. Scripts are named with a script path. For a script foo.xact directly in the script directory, the script path is just foo. For a script bar.xact in the xxx subdirectory the script path is xxx:bar.

The ABS view is used to specify absolute script paths. For example ABS.xxx:bar refers to the bar.xact script in the xxx subdirectory.

The REL view is used to specify relative script paths. For example REL.zap refers to the script zap.xact in the same directory as the invoking script.

The A function generates an html a element. For example @x:tag A[REL.foo]() { "to foo script" }. The foo.xact script must have a visible function with name foo.

Parameters of type string can be passed from one script to another. For example @x:tag A[REL.foo1]("test",false) { "foo1 with parameters" }. The foo1.xact script must have a visible function with name foo1 that takes two string parameters.

HTML form tags can be produced by calling function FORM instead of A. Form values are available in the target script via the special predefined passed value.

The A and FORM internally use the URLUnparseQuery function to produce the URL. There is also a build-in URLParseQuery that performs the inverse operation.
