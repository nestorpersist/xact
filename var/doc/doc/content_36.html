In Java classes it is common to have pairs of functions that are used as getters and setters for some property of instances of the class. @x:Act has a mechanism that allows these functions to share a single name and to use the same invocation syntax.

When a function call appears on the left-hand side of an assignment, then the lhs version of that function is called passing the value of the right-hand side of the assignment as an extra parameter. In other cases the rhs version of the function is called. The @x:func command allows options lhs and rhs.

The full richness of function call syntax also applies to lhs/rhs function pairs. This allows lhs dot qualification to be defined. If the rhs function has no parameters and no () formal parameter list then a "virtual" variable is defined that can be referenced from both lhs and rhs contexts.
