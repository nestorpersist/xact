Description

The lock type is is used for low-level multithread synchronization. It is based on the operations of the Java Object type. The @x:exclude command is used for mutual exclusion (or more generally to define a monitor) based on a specified lock object.

Type lock

@x:type lock

@x:type @x:func wait(%L)
@x:type @x:func notify()
@x:type @x:func notifyAll()
@x:type @x:func sys:Eq(a~lock,b~lock)~boolean
@x:type @x:func sys:Ne(a~lock,b~lock)~boolean
@x:self @x:func sys:Init()~void
@x:self @x:func toString()~string

Type Function wait

@x:func wait(%L)
%L There are three possible parameter forms. If there are no parameters then there is no timeout. If there is one parameter, then that int value will be a number of milliseconds for a timeout. If the are two parameters, then the first will be an int millisecond value and the second will be an int microsecond value that are added to form the timeout interval.

Semantics

This call must appear within a @x:exclude command for this lock. The current thread then waits until another thread calls notify or notifyAll for this lock.

If a timeout is specified, the wait will also end without any notify if the timeout interval has elapsed.

Type Function notify

@x:func notify()

Semantics

This call must appear within a @x:exclude command for this lock. If any other threads are waiting on this lock, then an arbitrary one is selected to be awakened. That thread will, however, not be able to proceed until the current thread also exits the enclosing @x:exclude command for this lock.

Type Function notifyAll

@x:func notifyAll()

Semantics

This call must appear within a @x:exclude command for this lock. If any other threads are waiting on this lock, then all are awakened. These threads will, however, not be able to proceed until the current thread also exits the enclosing @x:exclude command for this lock.

Type Function sys:Eq

@x:func sys:Eq(a~lock,b~lock)~boolean
a first parameter
b second parameter

Semantics

If a and b are the same lock, then the result is true, else the result is false.

Type Function sys:Ne

@x:func sys:Ne(a~lock,b~lock)~boolean
a first parameter
b second parameter

Semantics

If a and b are the same lock, then the result is false, else the result is true.

Self Function sys:Init

@x:func sys:Init()~void

Semantics

Initializes a new lock object.

Self Function toString

@x:func toString()~string

Semantics

The result is the string "lock".
