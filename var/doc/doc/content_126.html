Description

This command must be directly within an enclosing @x:fork command. It creates a new thread.

Command Syntax

'@x:thread' <id>

No options are permitted.

Each @x:thread command must appear immediately within a @x:fork command.

Name Semantics

The <id> thread name is declared the scope that surrounds the @x:thread command.

Execution Semantics

A new thread is created and the body is executed within that new thread. The result of that execution is discarded. Meanwhile the main thread continues executing after the @x:thread command. The result of executing the @x:thread command is null.

Examples


   @x:fork {
      @x:thread t1 { subtask1() }
      @x:thread t2 { subtask2() }
      maintask()
   }

See IncrParse example and queue example.
