Running as a CGI has the disadvantage that the Java JVM must be started every time a script is run. A faster alternative is to run the @x:Act interpreter as a servlet under a web server such as Apache Tomcat.

To run under Tomcat, you should copy two files into the Tomcat webapps directory tree.

  • On Windows, copy @x:Act file ...\xact\dev\win\web.xml into Tomcat file ...\webapps\xact\WEB-INF\web.xml. Next, you should edit this file to fix up file paths and optionally change interpreter options. Now, copy @x:Act file ...\xact\var\xact.jar into Tomcat file ...\webapps\xact\WEB-INF\lib\xact.jar.
  • On Macintosh, copy @x:Act file .../xact/dev/mac/web.xml into Tomcat file .../webapps/xact/WEB-INF/web.xml. Next, you should edit this file to fix up file paths and optionally change interpreter options. Now, copy @x:Act file .../xact/var/xact.jar into Tomcat file .../webapps/xact/WEB-INF/lib/xact.jar.

Finally you should (re)start Tomcat.

A typical URL for testing the servlet interface is given here.

127.0.0.1:8080/xact/xs?main
