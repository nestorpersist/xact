Description

This command is used to control name visibility.

Command Syntax

'@x:module' {<module specs>}?
<module specs> ::= {<module spec> ','}* <module spec>
<module spec> ::= 'visible'
<module spec> ::= 'hidden'
<module spec> ::= 'visible' '(' {<module item> ','}* <module item> ')'
<module spec> ::= 'hidden' '(' {<module item> ','}* <module item> ')'

No options are permitted.

<module item> ::= <id> {'[' ']'}?
<module item> ::= <id> {'[' ']'}? '>' <id>

At most one of hidden or visible (without any following list may be specified. No name may appear in both hidden visible name lists. Every name that appears within a name list must be defined within the @x:module command as visible.

Name Semantics

Select visible names declared within the @x:module scope that are visible are also declared within the scope that immediately contains the @x:module command.

There are several cases for each name that is visible within the module.

  • If there are no name lists and visible was specified, then the name is declared as visible in the enclosing scope.
  • If there are no name lists and visible was not specified, then the name is declared but not as visible in the enclosing scope.
  • If the name was specified in the visible name list, then the name is declared as visible in the enclosing scope.
  • If the name was specified in the hidden name list, then the name is declared but not as visible in the enclosing scope.
  • Names lists are present and the name does not appear in any name list and visible (with no list) was specified, then the name is declared as visible in the enclosing scope.
  • Names lists are present and the name does not appear in any name list and hidden (with no list) was specified, then the name is declared as but not as visible in the enclosing scope.
  • Names lists are present and the name does not appear in any name list and hidden (with no list) was not specified and visible (with no list) was not specified, then the name is not declared within the enclosing scope.

The name list forms also provide renaming. The name with the <id> specified before the > declared within the module is declared with the <id> specified after the > in the enclosing scope.

Execution Semantics

Execution of a @x:module command results in execution of its body. The result of that execution is discarded. The body scope instance is not discarded until the end of the enclosing scope instance. The result of executing a @x:module command is null.

Examples


   @module {
      @x:var v := 0;
      @x:func incr(),visible { v := v + 1 }
      @x:func val(),visible { v }
   }
   // can't see v here
   incr(); incr();
   val()
