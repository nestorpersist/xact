Syntax
<string> ::= '"' {{schar - '"'} | <escape>}* '"'
<string> ::= ''' {{schar - '''} | <escape>}* '''

Description

This form represents a string literal.

Semantics

The result of evaluating a string literal will have type string and the specified value. Escapes provide a way to include special characters. For example,
   "The next word is &quot;quoted&quot;."
represents the sentence: The next word is "quoted".

Escapes work by substituting in the value of the name. There are predefined names for common escapes quot, apos, amp, tab, nbsp, and eol. Names can also refer to arbitrary variables.
   @x:const x := "foo";
   @x:const y := 2 + 3;
   ~"&x;=&y;" == "foo=5"

It is illegal if an escape name is not defined at the place where the enclosing string occurs or if the value of the name does not have type string.

Parsing

Parsing a string literal without escapes results in a xdom:string tree node.

Parsing a string literal with escapes results in xdom:call node that represents a call to Cat. For example
   "abc&def;ghi"
and
   Cat("abc",def,"ghi")
after parsing are represented by identical trees except the first has a top-level xdom:call node with kind "double" and the second has a top-level xdom:call node with kind "normal". For single quotes
   'abc&def;ghi'
and
   Cat('abc',def,'ghi')
after parsing are represented by identical trees except the first has a top-level xdom:call node with kind "single" and the second has a top-level xdom:call node with kind "normal".
