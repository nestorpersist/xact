A new language is defined by writing a new @x:type that defines the language. This language type includes a Walk function that can perform the bind1, bind2, and exec passes. The pass to be performed is a parameter to Walk. The language type can optionally include self data that defines language specific interpretation state.

Before a language type can be used, a language object of that type must be created. The constructor for many language types take a single parameter that is the underlying language object. When such a language object encounters element with a tag that it does not know how to process that element is passed on to the underlying language object.

There are three standard language types: langtype:xact defines @x:Act commands (those starting with a x:), langtype:render handles command rendering, and langtype:error produces an error on every command.

The default language object for the @x:Act interpreter is langtype:xact(langtype:render()). The language type can be changed using the @x:lang command. The current language object is given by lang:current.

Definitions that are useful in writing and using language types can be found in the library:lang library.

If a language is being used in different threads, each will have its own language object. When a new thread is created the sys:Copy of the language object of the parent thread is called to create a language object copy for the child thread.

Every language type will have the following form. Note that [] type parameters can optionally also be present on language types. For a real example see the DocLang example.

Type myLangType

@x:type myLangType

@x:self @x:func sys:Init(under~thread)~void
@x:self @x:func sys:Copy()~myLangType
@x:self @x:func Walk(pass~enum["bind1","bind2","exec"],space~string,base~string,elem~xdom,lang,info~rec)

Self Function sys:Init

@x:func sys:Init(under~thread)~void
under the underlying language object. This parameter may be omitted on language types with no underlying language object such as langtype:render and langtype:error.

Semantics

Initializes the language object.

Self Function sys:Copy

@x:func sys:Copy()~myLangType

Semantics

Returns a copy of the language object for use in another thread.

Self Function Walk

@x:func Walk(pass~enum["bind1","bind2","exec"],space~string,base~string,elem~xdom,lang,info~rec)
pass the pass being performed.
space the name space of the command to be processed. This is usually the same as elem.space, but may be different during renaming.
base the name base of the command to be processed. This is usually the same as elem.base, but may be different during renaming.
elem the tree node command to be processed. Note that the tag of this node is ignored; space and base parameters are used instead.
lang The language object to be used for child elements. This permits two or more languages to be freely mixed. For example consider language object l1(l2()) where l1 handles a: tags and l2 handles b: tags. When processing a b: element l1 will pass the element on to the underlying l2 where the value of lang will be l1(l2()). This permits both a: and b: elements to appear as children as a b: element.
info Useful interpreter information that is passed in. In the bind1 pass, info.scope#rec will be the enclosing scope with type xdom:element.

Semantics

The result will be the pass result. For bind1 and bind2 this will be null. For exec this will be the result of execution.
