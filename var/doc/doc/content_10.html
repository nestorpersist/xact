@x:Act permits new abstract types to be defined.


@x:type stack[elem];
@x:self {
   @x:const data~array := array();
   @x:func isEmpty { data.size == 0 }
   @x:func push(b~elem)~void,visible {
      data.insert(b)
   }
   @x:func pop()~elem,~! isEmpty,visible {
      data(data.size);
      data.delete(data.size)
   }
   @x:func size~int,visible {
      data.size
   }
   @x:func get(i~int)~elem,~1 <= i & i <= size,visible {
      data(i)
   }
}

@x:const s~stack[boolean] := stack[boolean]();
s.push(true);
s.push(false);
"Start Dump"; eol;
@x:for i := 1..s.size {
   " "; i; "=>"; s.get(i); eol
}
"End Dump"; eol;
s.pop(); eol;
s.pop(); eol

The @x:type command defines data and operations associated with the type (in this case there are none). The immediately following @x:self command defines data and operations associated with instances of the type. Those that are marked visible can be seen by user of the type and its instances. Those not marked visible (in this case data and IsEmpty) are not visible outside.

Note the assertions on several of the functions. These are checked at the time the function is called.

Note that this is a parameterized type. The stack element type is passed in as elem.
