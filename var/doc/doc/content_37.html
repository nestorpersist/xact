In Java iterators are handled with iterator classes. @x:Act instead uses iterator functions declared with a @x:func command that includes an iterator option. Iterators are invoked by a @x:for command.

When @x:for command is executed, a new body function object is created and passed as the single formal parameter to the iterator function The body function has formal parameters that correspond to the iteration variables specified in the @x:for command and has a body that is the same as the body of the @x:for command. The iterators example shows a number of examples of iterators.

There are a number of predefined iterators as well as libraries of iterators.

The @x:Act iterator functions can be recursive. For example, its easy to write an iterator that walks a tree (something that would be difficult using Java iterators).

Iterators can have arbitrary name parameters. It is easy to compose iterators by having an iterator that takes other iterators as name parameters.

In Java it is easy to iterate over two data structures in parallel, looking at the first elements of each together, then the second elements, and so forth. The @x:Act Parallel iterator provides this feature. Internally it uses two threads, one for each of the two underlying iterators.
