@x:Act interpretation consists of four passes. The @x:Act interpreter calls these four passes in order.

  • The first pass is parsing. Parsing can be explicitly invoked by calling the Parse. Parsing is the same for all languages. These is no way to extend @x:Act syntax.
  • The second pass is processing scopes and name definitions. It also handles @x:use commands and handles visible and hidden options.
    • The third pass is processing name uses. The second and third passes can be invoked explicitly by calling the Bind function. Since name binding is language specific, a language object can be passed to Bind.
  • The fourth pass is execution. Execution can be explicitly invoked by calling the Exec function. Since execution is language specific, a language object can be passed to Exec.
