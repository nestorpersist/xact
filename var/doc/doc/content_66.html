There are two ways to invoke a script xxx.

  • As a script. Here the xxx.xact script is executed and its value is the result.
  • As a function. Here xxx.xact must contain a visible function with name xxx. Execution then has two steps. First, the script is executed and its result is discarded. Second the function xxx is called and its value is the result.

Execution as a function is the default. If these is no function with name xxx then execution as a script occurs.

The scriptOnly option will cause execution as a script even when there is a function with name xxx. The funcOnly option will cause execution as a function. Execution will fail if there is no xxx function.

When running @x:Act under a server as a CGI or servlet, the funcOnly option is normally specified for increased security.
