@x:Act provides the ability to define new languages and to write scripts that contain a mix of those languages. The namespace part of the tag of each command specifies which language to use.

By default @x:Act has two languages. All commands with the x: namespace are handled by the primary @x:Act language interpreter. All other commands are handled by the rendering interpreter.

New languages are defined in special types that tell how that language is to be interpreted.

The @x:lang command can be used to activate a language definition by specifying the language definition to use. All following commands within the same scope will be interpreted by that new language.

A language can take over the processing of all commands, but more frequently is set up so that multiple languages can be intermixed.
