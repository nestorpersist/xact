Function Dot

@x:func Dot(base,n~string,v~View)
@x:func Dot[%L](base,n~string,v~View)
%L name parameters
base value to be qualified
n component name
v (optional) type or view to use

Semantics

First the type or view, v1, to use must be determined.
  • If the v parameter is present, then let v1 be v.
  • Otherwise, if base?View, then let v1 be base.
  • Otherwise, let v1 be Type[base].
There are two major cases.
  • View. If base?View then
    • If there is a visible item with name n[L] within the top level of the @x:type or @x:view tag for v1, then that item is the result.
    • Otherwise the result is
         base.sys:Dot[n,L]#v1
  • Object. Otherwise,
    • If there is a visible item with name n[L] within the top level of the @x:self tag for v1, then that item is the result (with self set to base).
    • Otherwise the result is
         base.sys:Dot[n,L]#v1

Examples

Suppose T is a type, V is a view of that type, and X is an object with that type.
T.m1(a,b) Dot(T,"m1")(a,b) invokes a type method of T
X.m2(a,b) Dot(X,"m2")(a,b) invokes a self method of T (with ~self == base)
V.m3(a,b) Dot(V,"m3")(a,b) invokes a view method of V
X.m4#V(a,b) Dot(X,"m4",V)(a,b) invokes a self method of V (with ~self == base)

Suppose X has type xdom:element for the tag <foo a="3"><b c="x"/></foo>, then
X.tag the element tag ("foo")
X.a#rec a record component ("3")
X.b#bodyrec a body record component (<b c="x"/>)

To write a type method to handle T.xx and self method to handle X.yy
   @x:type T {
      @x:var x := 99;
      @x:func sys:Dot[a~enum["xx"]],visible { x + x }
   }@x:self {
      @x:var y := 50;
      @x:func sys:Dot[b~enum["yy"]],visible { y - 1 }
   }
