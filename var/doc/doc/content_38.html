@x:Act types are declared by a @x:type command followed by a @x:self command. The body of the @x:type command contains the declaration of variables and functions associated with the type (in Java these would be declared as static). The @x:self command contains the declaration of variables and and functions associated with objects of the type. For name binding purposes the @x:self tag is logically nested within the @x:type tag and can refer to names declared there.

An object of a type is created with a constructor. A constructor consists of a type followed by a set of parameters enclosed in () (note that unlike Java no new keyword is needed). This causes an instance of the @x:self scope to be created and all commands within it executed. If the special function with name sys:Init is declared within the @x:self command, it is then called passing it the parameters specified in the constructor. This instance lives until the object is garbage collected.

Variables and functions within the @x:type command that have option visible can be referenced by dot qualification of the type.

Variables and functions within the @x:self command that have option visible can be referenced by dot qualification of an object with the type. Within the @x:self command the name self is automatically declared and has the object as its value (self in @x:Act works just like this in Java).
