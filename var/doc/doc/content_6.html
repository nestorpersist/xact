Here is a simple @x:Act script for computing factorial.


@x:func fact(n) {
   @x:if n == 0 {
      1
   } @x:else {
      n * fact(n - 1)
   }
}
@x:for i := 1..10 {
   @x:const j := fact(i);
   "fact(&i;)=&j;&eol;"
}

When this script is run it produces the following output on STDOUT.

fact(1)=1
fact(2)=2
fact(3)=6
fact(4)=24
fact(5)=120
fact(6)=720
fact(7)=5040
fact(8)=40320
fact(9)=362880
fact(10)=3628800

@x:Act commands all start with @x:. The @ indicates the start of a command. This approach allows @x:Act to have no reserved words. The x: identifies a namespace (much as it does in XML). The x namespace is predefined for @x:Act commands. @x:Act allows other command sets to be defined, each with its own namespace. Note that the if-then-else involves two consecutive commands.

In the @x:Act language every expression and command has a value. Some commands such as @x:func and @x:var produce a null which is here discarded.

Expression prefix and infix operators are similar to those of Java with a couple of exceptions. Here := is the assignment operator (rather than the = used in Java).
