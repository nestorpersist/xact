Description

This special function should only be called within Language Definition types. Its assigns processing to some other language type. The DocLang example shows a sample use.

Function walk:other

@x:func walk:other(lang,space~string,base~string,elem~xdom,lang)
lang the language object that should take over processing.
space the name space of the command to be processed. This is usually the same as elem.space, but may be different during renaming.
base the name base of the command to be processed. This is usually the same as elem.base, but may be different during renaming.
elem the tree node command to be processed. Note that the tag of this node is ignored; space and base parameters are used instead.
lang The language object to be used for child elements. This permits two or more languages to be freely mixed. For example consider language object l1(l2()) where l1 handles a: tags and l2 handles b: tags. When processing a b: element l1 will pass the element on to the underlying l2 where the value of lang will be l1(l2()). This permits both a: and b: elements to appear as children as a b: element.

Semantics

The underlying language object is called and its result returned.
