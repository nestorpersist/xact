The lock type provides low-level thread synchronization and is based on Java monitors.

The @x:exclude command specifies a lock object and provides mutual exclusion based on that object. This is implemented as a Java synchronized block.

The lock methods wait, notify, and notifyAll are implemented using the corresponding Java Object methods.

The wait function can take one parameter (a time in milliseconds) or two parameters (a time in milliseconds and a time in nanoseconds) and will time-out after that interval is passed. If no parameters are specified, it will wait forever for a notify.

The queue example shows how locks can be used to implement the queue1 type which is equivalent to the build-in queue type.
