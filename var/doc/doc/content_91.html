Syntax
<int> ::= {digit}+

Description

This form represents an integer literal. There is no negative integer literal.

Semantics

The result of evaluating an integer literal will have type int and the specified value. A negative integer value can be produced by applying the - prefix operator to an integer literal.

Parsing

Parsing an integer literal results in a xdom:int tree node.

Examples


   15 - 40 == -25
