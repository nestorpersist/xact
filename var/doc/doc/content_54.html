@x:Act supports multithreading. The implementation uses Java threads. A "cactus-stack" model is used where each thread forks off a new branch of the @x:Act execution stack.

Threads are created using the @x:fork command. One or more @x:thread commands can appear directly within the body of the @x:fork command. Execution of a @x:thread command starts a new thread that executes the body of that command. Meanwhile execution of the body of the @x:fork command continues.

When the execution of the body of the @x:fork command is complete, it waits until execution of all the threads created within it are complete.

Special threads are also sometimes created by the Parse, Lines, and Chars functions.
