Views can not only be used to narrow the value set of a type, they can also be to expand a value set. In @x:Act (unlike Java), types do not automatically permit a null value. The following example shows a new view where values can be either strings or null.


@x:view StringNull {
   @x:func sys:Is(v),visible {
      v?string | v == null
   }
} @x:self {
   @x:func display()~string,visible {
      @x:if self == null {
         "***"
      } @x:else {
         self
      }
   }
}
~"abc"?string;
~! null?string;
~! "abc"?void;
~null?void;
~"abc"?StringNull;
~null?StringNull;
null.display#StringNull(); eol;
"abc".display#StringNull(); eol

Like types, operations can also be associated with view instances. Invoking the operation requires that the view be specified (following the #).

The special name self is automatically defined inside @x:self commands and refers to the current type or view instance. It plays the same role as this does in Java.
