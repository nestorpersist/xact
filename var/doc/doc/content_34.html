In Java, there can be multiple different versions of a single function which differ in the number and/or types of parameters. In @x:Act there can be only a single version of a function, but that single version can be set up to handle a wide range of parameters. This mechanism also supports optional keyword parameters and variable length parameter lists.

To write a function that has flexible parameter handling the function will have a single formal parameter prefixed with the % prefix operator.

When calling a function that has a single % formal parameter, the actual parameters are each evaluated and then combined into a single xdom:element object. The attributes of this object then contain the actual parameter values. Within the function body the rec view can be used to get named parameters and the array view can be used to get positional parameters.

There is also a special % form for actual parameters. If a function call contains a single actual parameter with is the % prefix operator applied to a xdom:element value then the actual parameter list is created by expanding the attributes of that xdom element.
